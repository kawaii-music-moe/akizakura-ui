# Spacing

余白（`margin` / `padding` / `gap`）は以下のスケールに沿って指定する。

## 基本スケール

CSS 変数として `:root` に定義する。単位は **rem**。

| トークン | 値 | 想定用途 |
|---|---|---|
| `--space-xs` | `0.25rem` | アイコンと文字の隙間など、極小の調整 |
| `--space-sm` | `0.5rem` | 標準的な「狭い」余白（リスト・カード内アイテム間） |
| `--space-md` | `1rem` | 標準的な「広い」余白（カードの padding、要素間） |
| `--space-lg` | `1.5rem` | コンテナや大きめエリアの padding |
| `--space-xl` | `2rem` | セクション間の区切り |

最頻出は `--space-sm` と `--space-md`。迷ったらこの 2 値のどちらかを選ぶ。

スケールはすべて `0.25rem` の倍数に乗っている。新しい値を追加するときも 0.25rem の倍数に揃える。

## em と rem の使い分け

- **rem（トークン経由）**
  ルート基準で絶対的に効かせたい余白に使う。コンテナの padding、セクション間の margin、レイアウト要素同士の gap など。原則ここで定義したトークンを用いる。
- **em（直書き可）**
  親要素の文字サイズに連動させたい余白に使う。見出し内のアイコンと文字の隙間、行内アイテムのインライン gap など、フォントサイズに比例して見えるべき箇所。`0.2em` `0.5em` などをその場で書いてよい。トークン化はしない。

## スケールから外れた値

特定レイアウトの位置合わせのみに必要な値（コンテンツのインデント幅、特定要素の整列オフセット等）はトークン化せず、その箇所に局所的に書いてよい。
ただし同じ値が複数箇所で同じ意味で繰り返し現れた時点で、専用トークンを切るかスケールに寄せるかを検討する。

## 運用ルール

- rem の余白は直接記述せず、可能な限りトークン経由で参照する。
- 新しい値を導入するときは、まずスケールに収まらないかを確認する。
- 0 は `0` のまま記述してよい（トークン化しない）。
