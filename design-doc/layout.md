# Layout

レイアウトの最上位方針。ここに書かれたルールは [card](card.md) / [list](list.md) などコンポーネント単位の規則よりも常に優先する。

## 徹底するルール

### 1. カードをネストさせない

カードの中にさらにカードを置かない。情報を階層化したくなった場合は、内側のカードを外して **見出し + 本文** として並べ直すか、別セクション・別ページに分割する。

カードは「ひとまとまりの情報」を表すコンテナであり、入れ子にすると「まとまりの粒度」が曖昧になり、見る側が階層を読み解く負担を負う。

### 2. 区切る線は最小限にする

罫線・枠線・下線は、**余白だけでは構造が読み取れない場合の最終手段** とする。情報の区切りは第一に [余白](spacing.md) で表現する。

線が増えるほど画面はノイジーになり、コンテンツそのものが線に埋もれる。「線を引きたい」と感じた瞬間に、まず「余白を増やして解決できないか」を考える。

### 3. 区切りは目ではなく並びから感じさせる

要素の配置・近接・余白で構造を伝える。ユーザーが「線が引いてあるから区切られている」と認識するのではなく、「並びを見るだけで自然にまとまりが分かる」状態を目指す。

近いものは同じグループ、遠いものは別グループ──この原則だけで多くの構造は表現できる。線や塗りはあくまで補助。

## 個別ルールへの影響

- [card](card.md): 「カードをネストしない」を最優先する。カード内にさらに枠で囲った下位ブロックを置きたくなった場合は、枠を外して見出し＋余白で表現する。
- [list](list.md): 区切りは「余白のみ」を第一候補とする。`border-bottom` は並びだけでは情報が読み取れない場合に限って使う。
