# Card

関連する情報をひとまとまりに見せる箱状のコンポーネント。

## 共通方針

- 内側の構造は縦方向の `flex` + `gap` で組み、子要素ごとに `margin` を積み上げない。
- 角丸は [`--radius`](shape.md) を既定とする。ページ最上位の囲いなど大きめのカードは `--radius-lg`。
- パディングは [spacing](spacing.md) のスケールに従う。標準は `--space-md`、密度を上げたい場合は `--space-sm`。
- 内部の要素間も `gap`（標準は `--space-md`）。

## 区切り方

カード自身の境界は次のいずれかで示す。

| 系統 | 使いどころ |
|---|---|
| 枠線（`border`） | 構造的に独立した存在として見せたいとき。1px を標準とし、特に強い独立性を持たせたい場合に限り太線（3px 程度）。 |
| 面（薄い背景塗り） | 一覧の中で軽くまとまりを示したいとき。[`--surface-tint`](color.md) を使う。 |

原則どちらか一方のみ。両方使うと情報量過多になる。

## 内部構造

- タイトル / 見出し → 本文 / 中身、の縦並びを基本とする。
- タイトルは [typography](typography.md) の見出しスケール（h2 / h3）を使う。
- 本文の段落の `margin` は 0 にし、間隔は親の `gap` で揃える。

## 影（box-shadow）

- 標準では使用しない。
- 一覧中で要素を軽く浮き上がらせたいときのみ、直下に薄くにじむ程度の弱い影を使う。
- 強い影で立体感を出すデザインは採用しない。
