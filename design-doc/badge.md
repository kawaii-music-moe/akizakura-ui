# Badge

状態やカテゴリを表す小さなラベル（ジャンルタグ、件数、ステータス表示など）。

## 共通方針

- 区切りは **塗り（面）** を基本とする。既定は [`--surface-tint`](color.md) の薄い面。
- 角丸は [`--radius`](shape.md) に揃える。pill（完全な丸）にはしない（[shape](shape.md) のトークン外の形状を増やさないため）。
- 文字は小さめ（0.8rem 目安）・`font-weight: 500`・1 行。
- アイコンを伴う場合は [icon](icon.md) に従い `em` の隙間で添える。

## 種類

| 種類 | 見た目 | 使いどころ |
|---|---|---|
| 既定（`.badge`） | `--surface-tint` の面 | 中立のラベル・ジャンルタグなど。 |
| アクセント（`.badge--accent`） | `--accent` の塗り | 特に目立たせたいラベル。 |
| 状態（`.badge--success` / `.badge--danger`） | 状態色の塗り | 成功 / エラーなどの状態表示。 |
| 枠線（`.badge--outline`） | 1px 枠線のみ | 面を使いたくない場面の控えめな表示。 |

塗り系は文字色を [`--bg`](color.md) にしてコントラストを確保する。

## 使い方の注意

- 1 箇所に多色のバッジを並べない。色は意味を持つので、ジャンル一覧などでは既定（中立）に統一し、強調が必要な 1 つだけ色を付ける。
- [list](list.md) の項目内や [card](card.md) のタイトル脇に添える使い方を基本とし、バッジ自体を主役にしない。
