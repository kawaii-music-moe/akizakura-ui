# Feedback

状態や確認を一時的に伝える、画面に「浮く」コンポーネント群（スナックバー・ダイアログ・ボトムシート）。

## 共通方針

- いずれも面が浮く必然性がある。akizakura では Material のような暗い反転面・強い elevation にはせず、**[`--bg`](color.md) + 枠線 + 弱い影（[`--shadow-card`](color.md)）** の浮遊サーフェスにする。
- 開閉・表示時間・ドラッグ等の**挙動は利用側 JS の責務**。CSS はその状態（開いている／閉じている）に応答するだけで、フレームワーク自体は JS 非依存を保つ（[color](color.md) のダークモードと同じ整理）。

## スナックバー / トースト

- 一時的なメッセージ。必要なら 1 つだけアクションを添える。
- アクションは [`--accent`](color.md) の文字ボタンにする（塗りを増やさない）。
- 画面下に浮かせる場合は `snackbar--fixed`。

## ダイアログ

- ネイティブ `<dialog>` 要素に当てる。`showModal()` で中央表示・フォーカストラップ・Esc 閉じが標準で効くため、独自実装しない。
- 背面の暗幕は `::backdrop` に [`--scrim`](color.md)。
- 角丸は [`--radius-lg`](shape.md)。構成は「タイトル → 本文 → アクション（右寄せ）」の縦積み。
- 閉じるボタンは `<form method="dialog">` で組むと JS なしで閉じられる。

## ボトムシート

- 下からせり上がる面。上端だけ大きめ角丸（[`--radius-lg`](shape.md)）、先頭に掴み handle を置く。
- 中身は [list](list.md) などをそのまま積む（[card](card.md) をネストしない）。
- 実際の出し入れ（`transform` アニメーション・背面 scrim）は利用側で行う。固定は `bottom-sheet--fixed`。
