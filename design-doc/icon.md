# Icon

## アイコンソースの優先順位

**Material Symbols Outlined（`GoogleIcon`）を最優先** で使う。Material Symbols に該当するアイコンが無い場合に限り、用途に応じて他のソースを当たる。

| 優先度 | ソース | コンポーネント | 補完する用途 |
|---|---|---|---|
| 1 | Material Symbols Outlined | `GoogleIcon` | 一般的な UI / 概念アイコン全般 |
| 2 | Fontello | `FontelloIcon` | Material Symbols に無いブランド・SNS アイコン（X / GitHub / SoundCloud 等） |
| 2 | Devicon | `Devicon` | Material Symbols に無い言語・技術スタックのロゴ |

新しいアイコンを導入するときは、まず Material Symbols を確認する。同等のアイコンが存在するなら他のソースに行かない。

## サイズ

- アイコンのサイズ指定には **`rem` または `em` のみ** を使う。`%` や `px` など他の単位は使わない。サイズ計算の連鎖で値が不定になるのを避けるため。
- インラインでテキストと並ぶアイコンは、**周囲のフォントサイズに視覚的に揃う**ことを目指す。各コンポーネントが内部で `em` ベースのスケーリングを行うため、利用側は基本的にサイズ指定をしない。
- アイコン単体を大きく見せたい場合（モバイルメニューのトグル、タップ領域確保など）のみ、利用箇所で `em` または `rem` で上書きする。上書き手段は `--icon-size` のように CSS 変数経由で渡す形を優先する。

## 色

- アイコンは原則として **周囲のテキスト色を継承** する。アイコン専用の色を割り当てない。
- ブランドロゴ（Devicon の言語ロゴ等）は固有色を持つことを許容する。

## アイコン + テキストの並び

- 並びは `display: flex; align-items: center;` を基本にする。
- アイコンと文字の隙間は親フォントサイズに連動させたい用途のため、`gap` を **`em`** で指定する（標準は `0.2em` 〜 `0.5em`）。詳細は [spacing](spacing.md) の em / rem 使い分けに従う。

## 装飾と意味

- 文字の頭に置くアイコンは「文字の意味を視覚で補う」位置付け。アイコン単独で意味を伝える使い方（ラベルなしのアイコンボタン等）をする場合は、適切なアクセシブル名を付与する。
