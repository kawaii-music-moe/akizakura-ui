# Navigation

画面間・セクション間を移動するためのナビゲーション群（アプリバー・下部ナビ・タブ）。スマホアプリの基本ナビを Material Design から借りつつ、見た目は akizakura の方針（線は最小・余白で構造・固定角丸）に翻訳する。

## 共通方針

- 背景は [`--bg`](color.md)。バー自身を面（塗り）で塗り分けたりはしない。
- 位置取り（`sticky` / `fixed`）は基底クラスに持たせず、`*--sticky` / `*--fixed` の派生で opt-in する。ナビ＝必ず固定、とは限らないため、レイアウトを押し付けない。
- 現在地は **accent 色** で示す。背景の反転や強い塗りでは示さない。

## アプリバー（上部）

- 構成は「先頭アクション → タイトル → 末尾アクション」。タイトルが伸び、アクションは両端に寄る。
- 区切りは下端の 1px のみ（[layout](layout.md): 線は最小）。影で持ち上げない。
- タイトルは [typography](typography.md) の見出しスケール相当（1.2rem / weight 600）。1 行に収め、溢れは省略する。

## 下部ナビ（bottom navigation）

- 項目は等幅で並べ、各項目は「アイコン上・ラベル下」の縦積み。
- 既定の項目色は [`--text-sub`](color.md)、現在地（`is-active` / `aria-current="page"`）は [`--accent`](color.md)。
- 区切りは上端の 1px のみ。

## タブ

- 現在タブは accent の下線（2px）で示す。これは「状態を色で伝えるための線の例外的な使用」（[layout](layout.md) / [notice](notice.md) と同じ整理）。
- タブ同士の区切り線は引かない。間隔は `gap` で取り、下端にだけコンテナの 1px を敷く。
- 横に溢れる場合はスクロールさせる（折り返さない）。

## 未決事項

- ナビゲーションドロワー（横から出るメニュー）は現時点で未規定。必要になったら別途定義する。
