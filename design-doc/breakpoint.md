# Breakpoint

レイアウトの分岐は **1 ブレイクポイント** のみで運用する。

| 名前 | 範囲 | 用途 |
|---|---|---|
| mobile | `width < 768px` | スマートフォン縦向きを想定 |
| desktop | `width >= 768px` | タブレット以上、PC を含む |

## 利用方法

### CSS

```css
/* desktop（768px 以上） */
@media (min-width: 768px) { ... }

/* mobile（767px 以下） */
@media (max-width: 767px) { ... }
```

`min-width` と `max-width` の境界は **768 / 767** で必ず噛み合わせる（範囲が重ならないようにする）。

### JavaScript / TypeScript

`document.body.offsetWidth` の境界として `768` を扱う。リサイズ追従が必要な箇所では、共通の判定（例: `isMobile` ストア）を経由して同じ値を参照する。

## 運用ルール

- ブレイクポイントは 768px の一点に固定する。グローバルなブレイクポイントを追加しない。
- 個別コンポーネント単位で中間サイズの調整が必要になっても、グローバルには持ち上げず、その要素の中で局所的に処理する。
